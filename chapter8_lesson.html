A graphics file contains digital photographs, line art, three-dimensional images, text data converted to images, and scanned replicas of printed pictures. Understanding graphics file types is essential for forensic investigators who need to locate, recover, and analyze image evidence.

Three Types of Graphics Files

Bitmap images are collections of dots, or pixels, in a grid format that form a graphic. Vector graphics are based on mathematical instructions to form lines, curves, text, and other geometric shapes. Metafile graphics are combinations of bitmap and vector images, offering flexibility for complex illustrations.

Programs for Graphics Files

Two main types of programs work with graphics: graphics editors (which allow creating and modifying images) and image viewers (which display images without editing capabilities). Forensic examiners use both types, but viewers are especially important for safely examining recovered evidence without modifying it.

Forensic Tip: No single viewer can read every graphics format. Maintain a library of multiple viewer programs to ensure you can open any format you encounter during an investigation.

Bitmap images store images as grids of individual pixels, while raster images store pixels in rows rather than a grid. Raster images are generally better for printing purposes because the row-based organization aligns with how printers render output.

Factors Affecting Image Quality

Several factors alter image quality: screen resolution (the density of pixels displayed onscreen), the software used to create or display the image, and the number of color bits used per pixel. Higher bit depths produce more colors and smoother gradients, but also result in larger file sizes.

Understanding Vector Graphics

Vector images have distinct characteristics: they use lines instead of dots, store only the mathematical calculations for drawing lines and shapes, have smaller file sizes compared to bitmap equivalents, and preserve quality when enlarged — a critical advantage over bitmap graphics that become pixelated when scaled up.

Key Distinction: In forensics, understanding whether an image is bitmap or vector affects how you analyze it. Bitmap images contain pixel-level data that can reveal editing traces, while vector files contain mathematical descriptions that may include embedded metadata about creation tools and timestamps.

Standard graphics file formats are common formats that most graphics programs and image viewers can open. These include .gif (Graphics Interchange Format), .jpeg/.jpg (Joint Photographic Experts Group), .bmp (Windows Bitmap), and .tif/.tiff (Tagged Image File Format).

Nonstandard Formats

Nonstandard graphics file formats are less common formats including proprietary formats, newer formats, and obsolete formats that most image viewers don’t recognize. Examples include .tga (Targa), .rtl, .psd (Adobe Photoshop), and .svg (Scalable Vector Graphics).

Raw File Format

The raw file format is typically found on higher-end digital cameras. The camera performs no enhancement processing — hence the term “raw.” This format maintains the best picture quality, but because it is a proprietary format, not all image viewers can display it. Raw files undergo demosaicing — the process of converting raw picture data to another format such as JPEG or TIFF.

EXIF Metadata

The Exchangeable Image File (Exif) format was developed by the Japanese Electronic Industry Development Association (JEIDA) as a standard for storing metadata in JPEG and TIFF files. Exif data can include camera make and model, GPS coordinates, date and time of capture, shutter speed, and other settings — all invaluable to forensic investigators.

Forensic Tip: Always examine Exif metadata in recovered image files. GPS coordinates can place a suspect at a specific location, and timestamps can establish a timeline of events. However, be aware that Exif data can be modified or stripped.

Most graphics file formats compress their data, including GIF and JPEG. Others, like BMP, rarely compress their data. Data compression is the process of coding data from a larger form to a smaller form. Graphics files and most compression tools use one of two compression schemes: lossless or lossy.

Lossless Compression

Lossless compression uses coding techniques for redundant bits to reduce file sizes without losing any information. The original data can be perfectly reconstructed. Key algorithms include Huffman coding and Lempel-Ziv-Welch (LZW). Formats using lossless compression include GIF and PNG. Compression tools include WinZip, PKZip, StuffIt, and FreeZip.

Lossy Compression

Lossy compression reduces file sizes by permanently discarding bits of information, which reduces image quality. The key algorithm is Vector Quantization (VQ), which uses an approach similar to rounding off decimal values to eliminate unnecessary data. JPEG is the most common lossy format. Lzip is an example of a lossy compression tool.

Forensic Alert: Lossy compression permanently destroys data. Each time a JPEG is re-saved, more information is lost. This is important when analyzing the quality and potential modification history of image evidence. Lossless formats preserve the original data completely.

Digital forensics tools can be used to analyze image files, including analyzing image headers and reconstructing fragmented image files. The key processes include identifying file fragments, repairing damaged headers, and carving data from unallocated space.

Identifying Graphics File Fragments

Before analyzing an image file, you may need to carve for file fragments. Carving (also known as salvaging outside North America) is the process of recovering file fragments scattered across a disk. Computer forensics tools can speed up this process significantly.

Repairing Damaged Headers

When file headers are damaged, you can identify image file headers from fragments found in slack or free disk space. The repair process involves: recovering more pieces of the file if needed, examining the file header and comparing it with a known good header sample, manually inserting correct hexadecimal values, and then testing the updated file.

Searching and Carving from Unallocated Space

Forensic tools like Autopsy can search and extract graphics data from unallocated disk space. The process involves searching for file signatures (magic numbers) in raw disk data and reconstructing complete files from the identified fragments.

Key Concept: File carving relies on identifying file headers (signatures) and footers in raw data. For example, JPEG files begin with the hex signature FF D8 FF and end with FF D9. Understanding these signatures is essential for successful data recovery.

Rebuilding File Headers

The main steps for rebuilding a damaged image file header include: recovering more pieces of the file if needed, examining the file header, comparing it with a good header sample, manually inserting correct hexadecimal values, and testing the updated file to confirm it opens properly in an image viewer.

Reconstructing File Fragments

The major steps for reconstructing fragmented files include: 1) Locate and export all clusters of the fragmented file. 2) Determine the starting and ending cluster numbers for each fragmented group of sectors. 3) Copy each fragmented group of sectors in their correct sequence to a recovery file. 4) Rebuild the corrupted file’s header to make it readable in a graphics viewer. 5) Add a .txt extension on all the copied sectors for initial analysis.

Analyzing Graphics File Headers with Hex Editors

You should analyze graphics file headers when you find files that your tools do not recognize. Use a hexadecimal editor to examine unknown graphics file headers. By comparing the hex values at the beginning of a file with known file signature databases, you can determine the actual file type regardless of its extension.

Forensic Tip: Suspects may rename file extensions to hide graphics files (e.g., renaming a .jpg to .doc). Always verify file types by examining the file header signature in a hex editor rather than relying on the file extension alone.

The Internet is the main source for finding explanations and viewers for uncommon or unknown image formats. Useful resources include file format databases and extension information sites. When keyword searches on recovered data produce false positives — results that contain the correct match but are not relevant to the investigation — additional analysis is needed.

Tools for Viewing Images

No single viewer can read every file format, so it is good practice to maintain many different viewer programs for investigations. Most GUI forensics tools include image viewers that display common formats like GIF and JPEG. For less common file formats, integrated viewers may identify the data as a graphics file or may not recognize it at all.

Best Practice: Maintain a comprehensive toolkit with multiple image viewers, hex editors, and format identification tools. Cross-reference findings across multiple tools to ensure nothing is missed during analysis.

Steganography is the art of hiding information inside image files. Unlike encryption (which makes data unreadable), steganography conceals the very existence of the hidden data. There are two major forms:

Insertion Steganography

With insertion steganography, the hidden data is not displayed when viewing the host file in its associated program. You need to analyze the data structure carefully to detect it. The hidden information is inserted into unused areas of the file format without significantly changing the visible appearance.

Substitution Steganography

Substitution steganography replaces bits of the host file with other bits of data. It usually changes the last two least significant bits (LSBs) of pixel values. The LSB is the lowest bit value in a byte. In Microsoft operating systems, bits are displayed from right to left, so the rightmost bit is the LSB. The most significant bit (MSB) is the highest bit value (the leftmost bit).

LSB Encoding Example

When substitution steganography modifies the LSBs of pixel bytes, the visual change is imperceptible to the human eye. For example, changing a pixel value from 10101010 to 10101001 alters the color by only 1 unit out of 255 — virtually invisible, but capable of encoding hidden messages across thousands of pixels.

Forensic Alert: When steganography has been done correctly, in most cases you cannot find the hidden data through visual inspection alone. Specialized steganalysis tools are required to detect variations in pixel data that indicate hidden information.

Using Steganalysis Tools

Steganalysis tools (or steg tools) are used to find information hidden in graphics images. This can be a very time-consuming task. Benefits of steganalysis tools include: detecting variations of an image, identifying the file format on a graphics file that has been renamed, and indicating whether a file contains an image.

Understanding Copyright Issues

Steganography has also been used to incorporate digital watermarks on image files for copyright purposes. As an investigator, you should be very careful when analyzing image files to avoid copyright conflicts.

Fair use allows using copyrighted material for noncommercial or educational purposes without having to compensate the material’s originator or owner. Important copyright considerations: there is no such thing as an international copyright law — copyright laws vary by country. Always check applicable laws in your jurisdiction.

Key Concept: Digital watermarking is essentially steganography used for legitimate purposes — embedding ownership information into images. As a forensic examiner, understanding both offensive steganography (hiding illicit data) and defensive steganography (copyright protection) is essential.

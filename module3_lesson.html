Data in a forensics acquisition tool is stored as an image file. There are three primary storage formats for digital evidence:

Raw Format

Raw format makes it possible to write bit-stream data to files. It creates simple sequential flat files of a suspect drive or data set.

AdvantagesDisadvantages
Fast data transfersRequires as much storage space as the original disk
Ignores minor data read errors on source driveTools might not collect marginal (bad) sectors
Most forensics tools can read raw formatDoes not contain metadata

Proprietary Formats

Features include the option to compress or not compress image files, capability to split images into smaller segmented files, and capability to integrate metadata into the image file.

Key Fact: The Expert Witness Compression format is currently the unofficial standard among proprietary formats for image acquisitions.

Disadvantages include the inability to share images between different tools and file size limitations for each segmented volume.

Advanced Forensics Format (AFF)

AFF was developed by Dr. Simson L. Garfinkel as an open-source acquisition format. Design goals include compressed or uncompressed image files, no size restriction, space for metadata, simple design with extensibility, cross-platform support, and internal consistency checks for self-authentication.

File extensions: .afd for segmented image files and .afm for AFF metadata.

There are two types of data acquisitions: static acquisitions (suspect drive is write-protected and can't be altered — repeatable) and live acquisitions (used when a suspect computer can't be shut down).

Four Methods of Data Collection

1. Disk-to-image file — Most common and flexible; creates a single image file or segmented files. 2. Disk-to-disk copy — Used when disk-to-image is not possible. 3. Logical disk-to-disk or disk-to-data file — Captures only specific files of interest. 4. Sparse data copy — Like logical acquisition but also collects fragments of unallocated (deleted) data.

Key Considerations

Size of the source disk and compression methods, whether you can retain the suspect's disk, the time available to perform the acquisition, and where the evidence is located are all important factors in choosing the right method.

Remember: A logical acquisition captures only specific files of interest. A sparse acquisition is similar but also collects fragments of unallocated (deleted) data.

To prevent damage or loss of evidence copies, consider these precautions:

1. Duplicate your evidence copies — Always have backups. 2. Use different tools — Acquire more than one image using different tools or techniques. 3. Consider host protected areas (HPAs) — Areas of a disk reserved for booting utilities and diagnostics, not visible to the OS. 4. Encrypted drives — Be prepared to deal with whole disk encryption, which performs sector-by-sector encryption of an entire drive.

Critical: Whole disk encryption makes copied data unreadable with a static acquisition method. You must plan for how to deal with encrypted drives before beginning your acquisition.

Windows-Based Acquisition

Windows-based tools make acquiring evidence more convenient, especially with hot-swappable devices. However, you must protect acquired data with a well-tested write-blocking hardware device, and these tools typically can't acquire data from a disk's host protected area.

Mini-WinFE Boot CDs and USB Drives

The Mini-WinFE Windows boot utility builds a forensic boot CD/DVD or USB drive with a Registry modification so connected drives are mounted as read-only.

Acquiring Data with a Linux Boot CD

Forensic Linux Live CDs contain utilities and are configured not to mount or to mount as read-only any connected storage media, eliminating the need for a separate write-blocker. Examples include CAINE, Kali Linux, Knoppix, and SANS Investigative Forensic Toolkit.

The dd Command

The dd ("dump data") command reads and writes data from a media device and creates raw format files. Its shortcomings: requires advanced skills and does not compress data. Combined with the split command, dd can segment output into separate volumes.

The dcfldd Command

dcfldd is the preferred Linux acquisition tool over dd. It offers additional capabilities: hex patterns for clearing disk space, error logging, multiple hashing options, progress display, segmented volumes, and verification of acquired data against the original.

FTK Imager Lite

FTK Imager Lite lets you view evidence disks and create disk-to-image copies at logical partition and physical drive levels. It can segment image files and image RAM on a live computer. The evidence drive must have a hardware write-blocking device or USB write-protection enabled.

Validating digital evidence is probably the most critical aspect of computer forensics. It requires using a hashing algorithm utility. Hashing algorithms range from CRC-32, MD5, and SHA-1 to SHA-512.

Linux Validation Methods

Use the md5sum and sha1sum commands to validate data acquired with dd. For dcfldd, use the hash option to designate an algorithm, hashlog to output results to a text file, and vf (verify file) to compare the image to the original medium.

Windows Validation Methods

Windows has no built-in hashing tools for digital forensics, but third-party utilities and commercial forensics programs have built-in validation features.

Important: Raw format image files don't contain metadata, so separate manual validation is recommended for all raw acquisitions.

RAID (Redundant Array of Independent Disks) is a computer configuration involving two or more disks originally developed as a data-redundancy measure. Size is the biggest concern because many RAID systems push into exabytes of data.

RAID LevelDescription
RAID 0Striping — data spread across disks for speed, no redundancy
RAID 1Mirroring — data duplicated on two disks for redundancy
RAID 5Striping with parity — distributed parity across all disks
RAID 6Double parity — survives two simultaneous disk failures
RAID 10Mirroring + Striping (RAID 1+0) combination

Key Acquisition Concerns

How much storage is needed? What RAID type is used? Are all drives connected? Do you have the right tool? Can the tool read a forensic RAID image or split data saves?

When a RAID system is too large for a static acquisition, use the sparse or logical acquisition method to retrieve only relevant data.

You can remotely connect to a suspect computer via a network connection and copy data. Remote acquisition tools vary in configurations and capabilities.

Drawbacks: Antivirus, antispyware, and firewall tools can interfere. Suspects with administrator rights could install security tools that alert them to remote access.

Available Remote Tools

ProDiscover Incident Response — Captures volatile system state, analyzes running processes, locates unseen files, remotely views IP ports, runs hash comparisons. Uses the PDServer remote agent in stealth mode.

EnCase Enterprise — Searches and collects across wide geographical areas, supports multiple OSs, performs triage, and runs simultaneous searches on up to five systems.

F-Response — A vendor-neutral remote access utility designed to work with any digital forensics program. Available in Enterprise, Consultant + Covert, Consultant, and Tactical versions.

R-Tools R-Studio — Designed for data recovery; creates raw format acquisitions and supports various file systems.

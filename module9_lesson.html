Examining and analyzing digital evidence depends on the nature of the investigation and the amount of data to process. Criminal investigations are limited to finding data defined in a search warrant, while civil investigations are often limited by court orders for discovery.

Understanding Scope Creep

Scope creep occurs when an investigation expands beyond the original description because of unexpected evidence found. Scope creep increases the time and resources needed to extract, analyze, and present evidence. Investigators must constantly evaluate whether new findings are within the original scope or require authorization to expand.

Refining the Investigation Plan

For private sector cases, it is critical to refine the investigation plan as much as possible. The investigation should be broad enough to encompass all relevant evidence, yet not so wide-ranging that time and resources are wasted. This balance is one of the most challenging aspects of digital forensics.

Key Principle: An investigator should maintain control of all evidence and findings and document everything as they progress through an examination. Documentation is your shield against challenges in court.

E-mail Harassment Cases

The approach to an e-mail harassment case depends on whether it is an internal corporate investigation or a civil/criminal investigation carried out by law enforcement. Corporate cases may be handled with less formal procedures, while law enforcement cases require strict chain-of-custody protocols.

Industrial Espionage Cases

An investigation of an employee suspected of industrial espionage can require the most work of any type of case. It is important to ensure the company has set up rules of use and limitations of privacy rights before beginning such an investigation. All suspected industrial espionage cases should be treated as criminal investigations.

Basic Investigation Steps

For all digital forensics investigations, an investigator should follow consistent steps: maintain control of evidence, document everything, establish chain of custody, and report findings to management and legal counsel as appropriate.

Important: Industrial espionage investigations require specialized staff including a digital investigator, a technology specialist knowledgeable about the compromised data, a network specialist for log analysis, and a threat assessment specialist (typically an attorney).

Supported Image Formats

Autopsy can analyze data from several sources, handling many formats including raw, Expert Witness (E01), and virtual machine image files (.vdi and .vhd). The utility includes an indexed version of the NIST National Software Reference Library (NSRL) of MD5 hashes.

NSRL Hash Sets

You can import NSRL reference hashes into Autopsy to filter out known good files (like operating system files) and identify known bad files. This dramatically speeds up the analysis process by allowing investigators to focus on files of interest.

Forensic Tip: The NSRL hash database helps investigators quickly filter thousands of known operating system and application files, allowing them to focus their analysis on unique or suspicious files that warrant detailed examination.

Validation is the process of ensuring the integrity of data to be presented as evidence in court. This is one of the most critical aspects of digital forensics. Without proper validation, evidence may be challenged or excluded.

Validating with Hexadecimal Editors

Advanced hexadecimal editors offer features not available in standard forensic tools, such as hashing specific files or sectors. Getting hash values with a hex editor can be faster and easier than with a full forensic suite. Investigators sometimes need the hash value of specific files or sectors to validate whether data fragments match known files.

Block-wise Hashing

The process of block-wise hashing builds a data set of hashes from sectors of an original file and examines sectors on the suspect’s drive to see whether any sectors match. If an identical hash value is found, you have confirmed that the file was stored on the suspect’s drive.

Known File Filter (KFF)

Some forensic tools include their own hashing databases. The Known File Filter (KFF) filters known program files from view and contains hash values of known illegal files, allowing comparison against files on the evidence drive.

Forensic Tip: Digital forensics tools have some limitations in performing hashing, so using advanced hexadecimal editors is necessary to ensure data integrity. Always use multiple validation methods to strengthen the defensibility of your evidence.

When creating forensic images in the Expert Witness or SMART format, additional options for hashing all the data are available. The resulting validation report lists both MD5 and SHA-1 hash values.

E01 Verifier

Autopsy includes a feature called the E01 Verifier for verifying Expert Witness image files. This tool recalculates the hash values stored in the E01 image header and compares them to confirm the image has not been altered since acquisition.

Hash Algorithm Comparison

AlgorithmOutput SizeUse in Forensics

Best Practice: Always generate both MD5 and SHA-1 (or higher) hash values during acquisition. Having two independent hashes makes it much harder for opposing counsel to challenge the integrity of your evidence.

Data hiding involves changing or manipulating a file to conceal information. Forensic investigators must be aware of multiple techniques suspects may use to hide evidence.

Common Data-Hiding Techniques

TechniqueHow It WorksDetection Method
Changing File ExtensionsRenaming .xlsx to .jpg to disguise file typeCompare file headers against extensions
Setting Hidden AttributesUsing OS settings to make files invisibleEnable hidden file viewing; use forensic tools
Hiding PartitionsUsing diskpart or similar utilitiesAccount for all disk space; use hex editors
Marking Bad ClustersPlacing data in clusters marked as badAccess bad clusters with disk editors
Bit-ShiftingChanges readable data to look like binary codeSpecialized analysis; check MD5 hash values
EncryptionUsing PGP, BitLocker, or similar toolsAttempt passphrase recovery; key escrow
Password ProtectionRequiring a password to access filesPassword cracking tools; dictionary attacks
Forensic Alert: Most advanced digital forensics tools check file headers and compare them against file extensions to identify potentially altered files. Always verify file types by examining the header signature rather than relying on extensions alone.

Hiding Partitions

A suspect can use the Windows diskpart utility to hide entire partitions. To detect this technique, an investigator should account for all disk space when examining an evidence drive. Most digital forensics tools or hexadecimal editors can access disk areas containing space that cannot be accounted for by the visible partitions.

Marking Bad Clusters

Another data-hiding technique used in FAT file systems is placing incriminating data in free or slack space on disk partition clusters. A suspect could use utilities to mark good clusters as bad, so the operating system will no longer access them. While this technique is no longer common, it can still be encountered.

Bit-Shifting

Bit-shifting changes data from readable code to data that looks like binary executable code. This technique can be verified by checking MD5 hash values of the original and shifted files. Some advanced malware uses bit-shifting to hide its malicious code from antivirus tools.

Key Concept: When examining an evidence drive, always compare the total physical capacity of the drive with the sum of all visible partitions. Any discrepancy may indicate a hidden partition that requires further investigation.

Steganography comes from the Greek word for “hidden writing.” It involves hiding messages in such a way that only the intended recipient knows the message is there. Steganalysis is the process of detecting and analyzing steganography files.

Digital Watermarking

Digital watermarking was developed as a way to protect file ownership. When used for steganography, watermarks are usually not visible to the naked eye.

Steganography Tools and Encryption

Many steganography tools are available as freeware or shareware and can insert information into a variety of files. If a suspect encrypts a plaintext file with PGP and then inserts the encrypted text into a steganography file, cracking the hidden message becomes extremely difficult.

Steganalysis Methods

MethodDescription
Stego-only AttackOnly the suspected stego-media is available for analysis
Known Cover AttackThe original cover-media and the stego-media are both available
Known Message AttackThe hidden message is known and can be compared
Chosen Stego AttackThe stego tool and stego-media are both known
Chosen Message AttackA known message is embedded using a specific tool to analyze patterns
Forensic Tip: When steganography has been done correctly, you typically cannot detect the hidden data through visual inspection alone. Specialized steganalysis tools that analyze statistical properties of pixel data are required.

Examining Encrypted Files

Programs such as PGP or BestCrypt can encrypt files so that without a password or passphrase, recovering the contents is extremely difficult. The technology known as key escrow is designed to recover encrypted data if passphrases are forgotten or corrupted.

The resources needed to crack encryption schemes are usually beyond what is available to small or medium organizations. Investigators should make an effort to persuade the suspect to reveal the encryption passphrase.

Rainbow Tables and Password Salting

A rainbow table is a file containing the hash values for every possible password that can be generated from a computer’s keyboard. Salting passwords adds extra bits to a password before it’s hashed, preventing rainbow tables from finding a matching hash value.

Remote Network Acquisition

You can remotely connect to a suspect computer via a network connection and copy data from it. Remote acquisition tools vary in configurations and capabilities, but all have drawbacks: antivirus, antispyware, and firewall tools can be configured to ignore remote access programs, and suspects could install their own security tools that trigger an alarm to notify them of remote access intrusions.

Security Warning: Remote acquisitions carry the risk that suspects with administrative rights could detect the intrusion and destroy evidence. Always assess whether the suspect has the technical capability to detect remote forensic tools before proceeding.

The cloud has introduced new ways of managing data that have created unique challenges for digital forensics investigators. New standards are being developed to improve security practices and incident responses in cloud environments.

History of the Cloud

The concept of cloud computing can be traced back to Professor John McCarthy of MIT and Dr. J.C.R. Licklider, director at the U.S. Department of Defense ARPA. In 1999, a CRM Web service was developed that applied digital marketing research to business subscribers, eventually paving the way for the modern cloud. Other early providers included Amazon EC2, Google Apps, Apple iCloud, and Microsoft OneDrive.

The NIST Definition

The National Institute of Standards and Technology (NIST) defines cloud computing as a computing storage system that provides on-demand network access for multiple users and can allocate storage to users to keep up with changes in their needs.

Key Concept: Cloud forensics applies digital forensics to cloud computing and is considered a subset of network forensics. It has three dimensions: organizational (structure of the cloud), legal (service agreements and jurisdictional matters), and technical (procedures and specialized forensics tools).

Understanding the different cloud service levels and deployment methods is critical because the type of cloud environment determines where evidence is stored and how it can be collected.

Three Service Levels

PaaS — Platform as a ServiceAn OS is installed on a cloud server; the customer loads their own applications and dataMost likely found on a desktop or server, or the remote provider’s infrastructure
IaaS — Infrastructure as a ServiceCustomers rent hardware and install whatever OSs and applications they needUsually found on a desktop or server; infrastructure may be owned by the company or the provider

Cloud Vendors

Major cloud service providers include Amazon EC2, Google Cloud Storage, Microsoft Azure, Oracle Cloud, IBM Cloud, Rackspace, and many others. Each CSP uses a variety of approaches and systems to build their cloud infrastructure, which means no two CSPs are configured in exactly the same way.

Because cloud computing is rapidly changing, laws have not kept up with the implications of storing files and data in locations that are not physically within reach. When investigating a cloud system, you must consider factors involving a CSP’s relationship with cloud users.

Service Level Agreements (SLAs)

Cloud service agreements (CSAs) — also called “master service agreements” or “service level agreements” — are contracts between a CSP and the customer describing what services are provided and at what level. CSAs should specify support options, penalties for services not provided, system performance, fees, and provided software or hardware.

CSAs should define the scope of services including: service hours, restrictions applied to customers, availability, support levels, response time for data transfers, throughput limitations, contingency plans for incident response, business continuity and disaster recovery plans, fees, security measures, and terminology. Forensics examiners should be most concerned with restrictions applied to customers and security measures.

CSP Documentation to Review

Document TypeDescription
PoliciesDetailed rules for internal operation including personnel responsibilities, management structure, and authorization to distribute information
StandardsGuidance for unique operations, hardware, and software describing obligations in daily operations and security

Jurisdiction Issues

There is currently no law that ensures uniform access or required handling procedures for the cloud. When data is commingled with other customers’ data, investigators must be careful to avoid privacy violations. The CSA can prescribe what laws are enforceable, but they don’t usually control privacy issues and criminal or civil procedures. Digital forensics examiners could be held liable when conducting cloud investigations and should consult with legal experts.

Important: If a search is announced, the risk of spoliation (failing to preserve evidence, whether intentional or through negligence) increases significantly.

When evidence needs to be seized, warrants are used in criminal cases and issued by law enforcement. When only information is needed, subpoenas are typically used in both civil and criminal cases.

ECPA Mechanisms

The Electronic Communications Privacy Act (ECPA) describes five mechanisms the government can use to obtain electronic information from a provider:

MechanismNotes
Search WarrantsCriminal cases only; requires law enforcement officer with probable cause
SubpoenasUsed in both civil and criminal cases
Subpoenas with Prior NoticeSubscriber or customer is notified in advance
Court OrdersJudicial authorization for specific data
Court Orders with Prior NoticeSubscriber or customer is notified in advance

In cloud environments, the property to be seized is usually data rather than physical hardware unless the CSP itself is the suspect. Government agency subpoenas, non-government and civil litigation subpoenas, and court orders each have their own requirements and limitations.

Cloud forensics procedures combine multiple disciplines: data recovery, network analysis to detect intrusions, database administration and security, software security, and international relations.

Anti-Forensics Countermeasures

Anti-forensics is the act of destroying electronically stored information (ESI) that could be potential evidence. Hackers may alter log records, modify MAC dates and times of important system files, and install malware to hide their activities. Calculating hash values of files and comparing them with known good files can help identify files that may have been altered.

Analysis Tip: Examining logs to compare Modified, Last Access, and Create (MAC) dates and times can reveal discrepancies. Metadata from affected files should always be examined to validate file accesses.

Methods used to collect evidence in cloud investigations depend on the nature of the case. Recovering deleted data from cloud storage might be limited by the type of file system the CSP uses.

Virtual Environment Snapshots

With cloud systems running in a virtual environment, snapshots can provide valuable information before, during, and after an incident. Forensic examiners should re-create separate cloud servers from each snapshot, acquire an image of each server, and calculate a hash for all files.

Acquisition Planning

Before initiating cloud investigations, always review the CSA to identify any restrictions that might limit collecting and analyzing data. The management plane — a tool with application programming interfaces (APIs) — allows reconfiguring a cloud on the fly and can be an important source of evidence about changes made to the system.

Provisioning (allocating cloud resources like additional disk space) and deprovisioning (deallocating cloud resources previously assigned to a user) activities should also be reviewed during the investigation, as they can indicate when users were added or removed from the system.

Investigating cloud incidents should involve the same systematic approach used in traditional digital forensics. The type of incident determines how to proceed with planning the investigation.

Investigating CSPs

When investigating a cloud service provider, ask these key questions:

• Does the investigator have authority to use cloud staff and resources?
• Is detailed knowledge of the cloud’s topology, policies, data storage methods, and devices available?
• Are there restrictions on collecting digital evidence from remote cloud storage?
• Is the data of interest local or remote? Can the CSP provide a forensically sound connection?
• For e-discovery demands on multitenant systems, is the data commingled with unrelated data?

Investigating Cloud Customers

When searching for evidence on a cloud customer’s computer or mobile device, check these locations:

• The Web browser’s cache file
• The CSP’s application folder, usually found under the user’s account folder
• The Windows Prefetch folder on the user’s computer

Understanding Prefetch Files

Prefetch files contain DLL pathnames and metadata used by an application. The OS reads the associated prefetch file and loads its information into the computer’s memory to speed up an application’s start time. Metadata includes the application’s MAC times in UTC format and a counter of how many times the application has run. If a prefetch file is deleted, Windows re-creates one that resets the MAC times and counter.

Three widely used cloud storage services — Dropbox, Google Drive, and OneDrive — each leave forensic artifacts on a user’s Windows computer. Understanding how each application stores data is critical for an investigator.

Cloud Storage Services

ServiceKey Forensic Details
DropboxOffers third-party applications such as e-mail, chat, and collaboration tools. Since 2012, uses base-64 format to store content, requiring specialized software to read. Registry entries reveal association.
Google DriveFree storage up to 15 GB. Application folder stored under the user’s account path. Registry entries and synchronized files provide forensic evidence.
OneDriveCreated by Microsoft (originally SkyDrive). Available with Windows 8 and later. User profiles stored in the user’s account path. Log files and synchronized files stored in various locations.

All three services have Registry entries that can reveal a user’s association with the cloud service, the files synchronized, and access timestamps. Users must maintain control over access to their cloud accounts for security purposes.

Forensic Tip: Specialized tools like Internet Evidence Finder (IEF) can decode Dropbox’s base-64 encoded content that standard forensic tools may not be able to read.

Several specialized tools and platforms can be applied to cloud forensic investigations. Each offers different capabilities for data collection, analysis, and evidence recovery in cloud environments.

Available Forensic Tools

ToolCapabilities
FROSTForensic Open-Stack Tools — integrates with OpenStack running in IaaS cloud environments. Adds forensics response capabilities for a CSP. A key feature is that it bypasses a virtual machine’s hypervisor.
F-ResponseA remote access tool that uses USB forwarding techniques to allow non-remote-capable forensics tools to access remote servers and their data storage. Requires F-Response Enterprise or Consultant plus KernelPro USB-Over-Ethernet.
Magnet AXIOM CloudA Cloud module that complements Process and Examine modules because most investigations now require collecting information from the cloud as well as from computers and mobile devices.
eDiscovery SolutionsEnterprise-level incident response and cybersecurity tools designed for large-scale cloud evidence collection and analysis.

Cloud Security Alliance (CSA)

The Cloud Security Alliance has developed resource documentation for CSPs and their staff to standardize cloud architectures for operating procedures, interoperability, testing, and validation. This effort helps create more consistent forensic environments across different cloud platforms.

Additional Resources: NIST Cloud Computing Program, SANS Cloud Security Fundamentals, and the Cloud Security Alliance all provide valuable references for cloud forensics practitioners.

Virtual machines have become common for both personal and business use. A forensics investigator needs to know how to analyze virtual machines and use them to analyze other suspect drives. The software that runs virtual machines is called a hypervisor.

Two Types of Hypervisors

Hypervisor TypeDescriptionExamples
Type 1 (Bare Metal)Loads directly on physical hardware; does not require a separate OS. Typically found on servers with large RAM and storage.VMware vSphere, Microsoft Hyper-V Server, XenServer, IBM PowerVM
Type 2 (Hosted)Rests on top of an existing OS. Usually found loaded on a suspect machine. Can run legacy hardware with specific OS requirements.VMware Workstation, VirtualBox, Parallels Desktop, KVM

Virtualization must be enabled in the BIOS before attempting to create a VM. Instruction sets called Virtual Machine Extensions (VMX) are necessary to use virtualization. Intel calls its CPU design for virtualization Virtualization Technology (VT).

Key Concept: Type 2 hypervisors are the ones most commonly found on suspect machines during forensic investigations. Type 1 hypervisors are typically found on enterprise servers and workstations.

An investigation involving virtual machines doesn’t differ much from a standard investigation. Begin by acquiring a forensic image of the host computer as well as network logs. By linking the VM’s IP address to log files, you may determine what websites the VM accessed.

Detecting Virtual Machines

To detect whether a VM is on a host computer, look for these clues:

Check the Users or Documents folder in Windows (or user directories in Linux), examine the host’s Registry for clues that VMs have been installed or uninstalled, and look for the existence of a virtual network adapter.

Standard Investigation Procedure

Follow a consistent procedure: 1) Image the host machine. 2) Locate the virtualization software and VMs using file extensions and network adapters. 3) Export from the host machine all files associated with VMs, including log files, virtual adapters, and snapshots. 4) Record hash values of all associated files. 5) Open a VM as an image file in forensics software and create a forensic image, or mount the VM as a drive and then image it, or do a live search.

The Importance of Snapshots

Snapshots record the state of a VM at a particular moment (recording only changes in state, not a complete backup). Live acquisitions of VMs are often necessary because they include all snapshots. When acquiring a static image of a VM file, snapshots might not be included.

Important: Most forensics tools can mount VMs as an external drive. By mounting a VM as a drive, you can make it behave more like a physical computer. Another method is making a copy of the forensic image and opening it while running, though this alters evidence and may be considered nonstandard.

Having a good working relationship with network administrators and lead technicians can be helpful in most investigations involving Type 1 hypervisors, as these systems are typically managed by IT departments.

Common Type 1 Hypervisors

HypervisorVendorNotable Features

Live acquisitions are especially useful when dealing with active network intrusions or attacks. Live acquisitions done before taking a system offline are becoming a necessity because attacks might leave footprints only in running processes or RAM.

Order of Volatility (OOV)

The order of volatility (OOV) determines how long a piece of information lasts on a system. RAM and running processes might last only milliseconds; items stored on hard drives can last for years. Always capture the most volatile data first.

General Steps for a Live Acquisition

1) Create or download a bootable forensic CD or USB drive. 2) Keep a detailed log of all your actions. 3) Use a network drive as an ideal place to send collected information. 4) Copy the physical memory (RAM) first. 5) The next step varies depending on the incident being investigated. 6) Get a forensic digital hash value of all files recovered during the live acquisition.

Live Acquisition Tools

Several tools are available to capture RAM. GUI tools are easy to use but often require a lot of system resources and might get false readings in Windows OSs. Command-line tools give you more control.

ToolTypeDescription
Mandiant MemoryzeGUIMemory forensic software for imaging and analysis
Belkasoft RamCapturerGUILightweight RAM capture utility
Kali LinuxCLI/GUIUpdated version of BackTrack; full forensics distribution
Sleuth KitCLICommand-line forensic analysis tools
Critical: Live acquisitions don’t follow typical forensics procedures. You are interacting with a running system, which inherently changes its state. Document every action meticulously and always capture volatile data before less volatile data.

Network forensics is the process of collecting and analyzing raw network data and systematically tracking network traffic to ascertain how an attack was carried out or how an event occurred on a network.

Intruders leave a trail behind. Knowing your network’s typical traffic patterns is important in spotting variations in network traffic. This can also help you determine whether a network is truly under attack or experiencing normal anomalies.

The Need for Established Procedures

Taking the time to follow standard procedures is essential to ensure that all compromised systems have been found. Procedures must be based on an organization’s needs and should complement the network infrastructure.

Developing Procedures for Network Forensics

The standard procedure for network forensics: 1) Always use a standard installation image for systems on a network. 2) Close any way into a network after an attack. 3) Attempt to retrieve all volatile data. 4) Acquire all compromised drives and make forensic images. 5) Compare files on the forensic image to the original installation image.

Key Difference: Network forensics differs from digital forensics in that network evidence is often transient and volatile. While a hard drive preserves data even after power loss, network traffic exists only momentarily unless actively captured and logged.

The layered network defense strategy sets up layers of protection to hide the most valuable data at the innermost part of the network.

Defense in Depth (DiD)

Defense in depth (DiD) is a similar layered approach developed by the NSA that focuses on three modes of protection:

ModeDescription
PeopleTraining, awareness, and security policies for all personnel
TechnologyHardware and software security measures (firewalls, IDS, encryption)
OperationsProcedures, monitoring, and incident response processes

Testing networks is as important as testing servers. You need to be up to date on the latest methods intruders use to infiltrate networks as well as methods internal employees use to sabotage networks.

Best Practice: A robust defense-in-depth strategy doesn’t rely on any single security measure. Instead, it layers multiple controls so that if one fails, others continue to protect the network. Think of it like concentric castle walls.

Network logs record ingoing and outgoing traffic and are available from network servers, routers, and firewalls. The tcpdump command-line program and Wireshark are common tools for examining network traffic.

Network Traffic Analysis

Analyzing traffic includes creating top 10 lists of most active IP addresses, looking for patterns to understand the nature of an attack, and determining if other companies are involved. If other companies are involved, you should proceed as in a digital forensic investigation and contact those companies to enlist their aid.

Network Management Tools

ToolPurpose
SplunkLog management, security analytics, and event monitoring
SpiceworksNetwork monitoring and IT management
NagiosInfrastructure monitoring and alerting
CactiNetwork graphing and performance monitoring
Security Warning: Understand the risks of letting attackers use these network tools with administrative rights. If an attacker gains access to tools like Splunk or Nagios with admin privileges, they can monitor your response efforts and cover their tracks.

Packet analyzers are devices or software that monitor network traffic. Most work at layer 2 or 3 of the OSI model. Most packet analyzer tools can read anything captured in the Pcap (packet capture) format.

Some packets can be identified by examining the flags in their TCP headers. Understanding these flags (SYN, ACK, FIN, RST, PSH, URG) is essential for analyzing network forensic evidence.

Common Packet Analyzer Tools

ToolDescription
tcpdumpCommand-line packet capture and analysis

TcpsliceExtracts portions of tcpdump trace files

EtherapeGraphical network traffic monitor

Forensic Tip: Wireshark is the most widely used packet analyzer in network forensics. It can capture live traffic, read previously saved captures, filter packets by protocol or content, and reconstruct TCP streams to view full conversations between hosts.

Virtual Network Investigations

When investigating a virtual network, you need to modify your approach compared to physical networks. A virtual switch is different from a physical switch in that there’s no spanning tree between virtual switches. An additional complication is that hypervisors can assign MAC addresses to virtual devices, and devices can have the same MAC address on different virtual networks.

Things get even more complex when dealing with a cloud service provider (CSP) that hosts networks for hundreds of companies. Wireshark and Network Miner are two tools capable of analyzing virtual networks.

The Honeynet Project

The Honeynet Project was developed to make information widely available in an attempt to thwart Internet and network attackers. Its objectives are awareness, information, and tools.

A honeypot is a normal-looking computer that lures attackers to it. Honeywalls monitor what’s happening to honeypots on your network and record what attackers are doing.

Major Network Threats

ThreatDescription
DDoS AttacksHundreds or thousands of compromised machines (zombies) flood a target with traffic
Zero-Day AttacksAttackers exploit vulnerabilities before patches are available
IP SpoofingAttackers falsify the source IP address to disguise their identity
Session HijackingAttackers take over an authenticated session between two hosts
Key Concept: A honeypot looks like a real target but is actually a trap. Honeywalls surrounding honeypots capture attacker techniques without exposing real systems. The Honeynet Project shares this intelligence globally to help organizations defend their networks.
